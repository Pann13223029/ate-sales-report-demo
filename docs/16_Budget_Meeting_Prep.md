# Budget Approval Meeting — Question Prep

> **Meeting:** Budget approval for Vercel Pro + Google Cloud migration
> **Attendees:** You (developer), Sales Manager, IT team member
> **Goal:** Get approval for 700 THB/month + company card authorization

---

## Questions from the Sales Manager

---

**Q: ทำไมต้องจ่ายเงิน ก่อนหน้านี้ก็ใช้ฟรีได้?**
*Why pay when it was free before?*

A: ตอน demo ใช้ฟรีได้เพราะเป็นบัญชีส่วนตัว ทดลองใช้ แต่พอเป็นงานจริงของบริษัท ข้อกำหนดของ Vercel บอกชัดว่าแพลนฟรีห้ามใช้ commercial ถ้าเขาตรวจเจอ ปิดระบบได้ทันทีโดยไม่แจ้งล่วงหน้า เซลล์ส่งรายงานมาแล้วบอทไม่ตอบ ข้อมูลหายเลย — 700 บาทเหมือนซื้อประกันให้ระบบทำงานได้ต่อเนื่อง

---

**Q: 700 บาทต่อเดือน ถ้ารวมค่าอื่นๆ จริงเท่าไหร่?**
*What's the real total cost?*

A: ตอนนี้ 700 บาทจริงๆ ครับ ที่เหลือทั้งหมดฟรี — LINE, Google Sheets, Dashboard, AI ทั้ง Google Gemini และ Groq ในอนาคตถ้าต้อง upgrade AI เป็น paid tier ค่าใช้จ่ายจะเพิ่มอีกประมาณ 35-200 บาท/เดือน ซึ่งจะแจ้งก่อนเสมอ สูงสุดไม่เกิน 900 บาท/เดือน

---

**Q: เซลล์ต้องเปลี่ยนวิธีใช้งานอะไรมั้ย?**
*Does anything change for the reps?*

A: ไม่เปลี่ยนเลยครับ เซลล์ใช้งานเหมือนเดิมทุกอย่าง — ส่งข้อความใน LINE เหมือนเดิม บอทตอบเหมือนเดิม ข้อมูลขึ้น Sheet เหมือนเดิม สิ่งที่เปลี่ยนคือ "ข้างหลัง" — ระบบจะเสถียรขึ้น เร็วขึ้น และอยู่ภายใต้บัญชีบริษัท

---

**Q: บอทช่วยอะไรทีมเซลล์ได้จริง? มีตัวเลขมั้ย?**
*What's the actual value for my team?*

A: ก่อนมีบอท เซลล์ต้องกลับมากรอกข้อมูลเอง ข้อมูลไม่ครบ ไม่ทันเวลา ดีลหลุดเพราะไม่มีคน follow up ตอนนี้:
- เซลล์รายงานได้ใน 10 วินาที (แค่พิมพ์ใน LINE)
- ทุกดีลมี timestamp ชัดเจนว่าอัพเดทล่าสุดเมื่อไหร่
- ดีลที่ไม่มีอัพเดท 7 วัน ระบบแจ้งเตือนอัตโนมัติทุกวันจันทร์
- Dashboard เห็น pipeline ทั้งหมดแบบ real-time
- ถ้าเซลล์ลาออก ข้อมูลลูกค้าอยู่ในระบบ ไม่หายไปกับคน

---

**Q: ถ้าอนุมัติแล้ว ต้องทำอะไรฝั่งเซลล์มั้ย?**
*Do I need to do anything?*

A: ขอให้พี่ช่วยเรื่องเดียวครับ — ตอนเปิดใช้ระบบจริง ช่วยเป็นคนส่งข้อความแรกให้ทีมเห็น แล้วบอกทีมในที่ประชุมสั้นๆ 15 นาทีว่าเริ่มใช้แล้ว ไม่ต้อง training อะไรเลย เซลล์แค่ส่งข้อความใน LINE เหมือนที่ทำอยู่ ถ้าพี่เป็นคน champion ทีมจะเชื่อมั่นและใช้งานเร็วขึ้นมาก

---

**Q: ถ้าบอทตอบผิดล่ะ? AI เข้าใจภาษาไทยได้ดีแค่ไหน?**
*What if the AI gets it wrong?*

A: AI เข้าใจภาษาไทยได้ดีมากครับ ใช้ Google Gemini 2.5 Flash ซึ่งเก่งภาษาไทยอันดับต้นๆ และมีตัวอย่าง 8 แบบสอนไว้แล้ว (เยี่ยมลูกค้า, โทร, ส่ง QT, ปิดดีล, เสียงาน, ประมูล, ส่งซ่อม, ไม่ใช่เรื่องขาย) ถ้าตอบผิดเซลล์เห็นทันทีจากข้อความยืนยัน แล้วใช้คำสั่ง "อัพเดท MSG-XXXXXXXX" แก้ไขได้เลย ช่วง 1-2 สัปดาห์แรกผมจะคอย monitor ทุกข้อความ ถ้ามี pattern ที่ AI เข้าใจผิดบ่อยจะเพิ่มตัวอย่างสอนเข้าไปครับ

---

**Q: เซลล์ส่งรูปได้มั้ย? เช่น รูป PO หรือนามบัตร**
*Can reps send photos?*

A: ตอนนี้ถ้าส่งรูปมา บอทจะตอบกลับว่ารับได้แค่ข้อความตัวอักษร แล้วแนะนำให้พิมพ์สรุปสั้นๆ แทน ในอนาคตวางแผนจะเพิ่ม OCR อ่านรูป PO/นามบัตรได้ แต่ตอนนี้ขอเป็นข้อความก่อนครับ

---

**Q: Dashboard ใครดูได้บ้าง? เซลล์เห็นของคนอื่นมั้ย?**
*Who can see the dashboard?*

A: Dashboard มีแค่ผู้บริหารเท่านั้นที่ดูได้ครับ เซลล์จะไม่เห็น Dashboard จาก Rich Menu มีแค่ปุ่ม "วิธีรายงาน" "วิธีอัพเดท" กับ "เปิด Sheets" — Sheets เซลล์ดูได้แต่จะเห็นข้อมูลของทุกคนเหมือนกัน ถ้าอยากจำกัดเพิ่มก็ทำได้ครับ

## Questions from IT

---

**Q: ระบบ host อยู่ที่ไหน? ใครดูแล server?**
*Where is it hosted? Who manages the server?*

A: อยู่บน Vercel (serverless) ไม่มี server ให้ดูแลเลยครับ ไม่ต้อง patch OS, ไม่ต้องดู disk space, ไม่ต้อง restart service — Vercel จัดการให้หมด auto-scale ได้เอง code อยู่บน GitHub push แล้ว deploy อัตโนมัติ ถ้าต้อง rollback กดปุ่มเดียวใน Vercel dashboard กลับ version เดิมได้ทันที

---

**Q: Security เป็นยังไง? ข้อมูลลูกค้าปลอดภัยมั้ย?**
*How secure is it?*

A: มีหลายชั้นครับ:
- Webhook จาก LINE ตรวจลายเซ็น HMAC-SHA256 ทุกครั้ง ข้อความปลอมเข้าไม่ได้
- มี user allowlist เฉพาะเซลล์ 11 คนที่อนุญาตเท่านั้นที่ใช้ได้
- ข้อมูลที่เขียนลง Sheet มีการป้องกัน formula injection
- API key ทุกตัวเก็บเป็น encrypted environment variable
- Error message ไม่เคย leak ข้อมูลภายในออกไป
- Log ไม่เก็บชื่อเซลล์หรือข้อมูลลูกค้า ใช้แค่ pseudonymized ID

---

**Q: ถ้าระบบล่มจะรู้ได้ยังไง?**
*How do we know if it goes down?*

A: ตอนนี้ยังไม่มี alerting อัตโนมัติครับ แต่ในแผนจะเพิ่มภายใน 2 สัปดาห์หลัง go-live:
- Daily health check ผ่าน GitHub Actions — ถ้าบอทไม่ตอบจะแจ้งเตือนใน LINE group
- ถ้า weekly stale check cron fail ก็มี alert
- จะต่อ Axiom (ฟรี) เป็น log monitoring ดู error rate ได้
- Vercel Pro มี function logs เก็บยาวขึ้นจาก 1 ชั่วโมงเป็น 3 วัน

---

**Q: ถ้าต้อง maintain ใครทำ? ถ้าคนทำออกล่ะ?**
*Who maintains this? What if the developer leaves?*

A: ตอนนี้ผมคนเดียวครับ แต่พอย้ายมาบัญชีบริษัทแล้ว:
- GitHub org ของบริษัท — เพิ่มคนเข้ามาดูแลได้
- Vercel team account — IT เข้า dashboard ดู log, deploy ได้
- Google Cloud project ของบริษัท — rotate key ได้เอง
- มี ARCHITECTURE.md อธิบายระบบละเอียด 16 sections
- มี Production Migration Plan อธิบายทุกขั้นตอน
- code ทั้งหมดอยู่ใน 1 ไฟล์หลัก (webhook.py) Python developer คนใหม่ read แล้วเข้าใจได้

ผมแนะนำให้เพิ่มอีก 1 คนจาก IT ที่มี admin access ด้วยครับ เผื่อกรณีฉุกเฉิน

---

**Q: Vercel Pro $20 เป็นรายเดือนหรือรายปี? ยกเลิกได้มั้ย?**
*Is it monthly or annual? Can we cancel?*

A: รายเดือนครับ ตัดบัตรอัตโนมัติทุกเดือน ยกเลิกได้ทุกเมื่อ ไม่มีสัญญาผูกมัด ถ้ายกเลิกจะกลับเป็นแพลนฟรี (แต่จะผิด ToS ถ้ายังใช้งาน commercial อยู่)

---

**Q: Google Cloud บอกฟรี แต่จะโดนชาร์จตอนไหน?**
*When does Google Cloud start charging?*

A: Google Cloud มี free tier ที่ครอบคลุมการใช้งานของเราครับ — Sheets API, service account, ไม่มีค่าใช้จ่าย จะมีค่าใช้จ่ายก็ต่อเมื่อ:
- อัพเกรด Gemini AI จาก free tier เป็น paid (ถ้า Google ลด free quota ในอนาคต) — ประมาณ 35-105 บาท/เดือน
- หรือเพิ่ม feature ใหม่ที่ต้องใช้ Google Cloud service เพิ่ม

ทุกกรณีจะแจ้งให้ทราบก่อนเสมอ ไม่มีค่าใช้จ่ายเซอร์ไพรส์ ตอนสร้าง project ใหม่แนะนำให้ตั้ง billing alert ไว้ที่ 500 บาท/เดือน เพื่อความสบายใจครับ

---

**Q: ทำไมไม่ใช้ server ของบริษัทเอง?**
*Why not use our own servers?*

A: Serverless ไม่ต้องดูแล server เลยครับ ไม่ต้อง patch, ไม่ต้อง monitor disk, ไม่ต้องจ่ายค่า server ตลอด 24 ชม. ถ้าไม่มีคนใช้ก็ไม่เสียเงิน ถ้าเซลล์ 11 คนส่งพร้อมกัน Vercel scale ให้อัตโนมัติ ไม่ต้องมีคน on-call ดู server ตอนดึก ค่า server เอง + คน maintain จะแพงกว่า 700 บาท/เดือนมากครับ

---

**Q: Data อยู่ต่างประเทศ ผิดกฎหมายมั้ย?**
*Data is stored overseas — is that legal?*

A: ถูกกฎหมายครับ PDPA มาตรา 28 อนุญาตให้ส่งข้อมูลไปต่างประเทศได้ ถ้ามีมาตรการคุ้มครองที่เหมาะสม ซึ่งเรามี:
- Google มี DPA (Data Processing Agreement) มาตรฐาน
- Vercel Pro มี DPA (เหตุผลหนึ่งที่ต้อง upgrade)
- ข้อมูลเข้ารหัสทั้งตอนส่ง (HTTPS) และตอนเก็บ (Google encryption)
- ผมเตรียมเอกสาร PDPA compliance ไว้แล้วครบ — Privacy Policy, ROPA, Data Retention Policy, Breach Response Plan

---

**Q: PDPA เรื่องใหญ่จริงมั้ย? มีบริษัทไหนโดนปรับจริงหรือยัง?**
*Is PDPA actually enforced? Has anyone been fined?*

A: ช่วง 2024-2025 เริ่มมีการบังคับใช้จริงแล้วครับ โดยเฉพาะกลุ่มธนาคารและโทรคมนาคม ถึงบริษัท B2B อุตสาหกรรมอย่างเรายังไม่ใช่เป้าหลัก แต่ถ้าลูกค้าร้องเรียน 1 ราย PDPC ก็สอบสวนได้ ตอนนั้นถ้าเราไม่มีเอกสาร ไม่มี DPA กับผู้ให้บริการ จะอธิบายยากมาก โทษปรับสูงสุด 5 ล้านบาท/กรณี — 700 บาท/เดือนถูกกว่าค่าปรับมากครับ

## Questions They Might Ask Each Other (prepare for these)

---

**Sales Manager → IT: "ระบบนี้ IT ต้องดูแลเยอะมั้ย?"**

ตอบแทน IT ได้: ไม่เยอะครับ serverless ไม่มี server ดูแล ผม maintain code อยู่แล้ว IT แค่ต้องมีคน 1 คนที่มี admin access เผื่อกรณีฉุกเฉิน ประมาณ 2 ชั่วโมง/เดือนสำหรับ monitoring กับ key rotation

---

**IT → Sales Manager: "ทีมเซลล์ใช้จริงมั้ย? ถ้าลงทุนแล้วไม่มีคนใช้?"**

ให้ Sales Manager ตอบ (ถ้าเขา champion อยู่): ทีมใช้อยู่แล้วจาก demo phase ถ้ายังไม่แน่ใจ บอกว่าจะเริ่มจาก 2-3 คนก่อน ถ้า 2 สัปดาห์มี adoption 70%+ ก็ยืนยันว่าคุ้ม ถ้าไม่ work ยกเลิก Vercel Pro ได้ทันที ไม่มีสัญญาผูก

---

**ทั้งคู่: "มี option อื่นที่ถูกกว่ามั้ย?"**

A: ผมเปรียบเทียบแล้วครับ:
- Self-hosted server: ค่า VPS 500-2,000 บาท/เดือน + ต้องมีคน maintain 24/7 — แพงกว่าและเสี่ยงกว่า
- ใช้ platform อื่น (AWS Lambda, Google Cloud Run): ค่าใช้จ่ายใกล้เคียง แต่ setup ซับซ้อนกว่า ระบบออกแบบมาสำหรับ Vercel โดยเฉพาะ ย้ายไป platform อื่นต้อง rewrite
- ไม่ใช้ระบบเลย กลับไปกรอก Excel: ฟรีแต่ไม่มีใครทำ ข้อมูลไม่ครบ ดีลหลุด
- 700 บาท/เดือนเป็น option ที่ถูกที่สุดที่ทำงานได้จริงครับ

## Meeting Tips

- **อย่าเริ่มด้วยเทคนิค** — เริ่มด้วยปัญหาธุรกิจ (PDPA risk, bus factor) แล้วค่อยลงเทคนิคถ้าถูกถาม
- **ให้ Sales Manager พูดเรื่อง value** — ถ้าเขา champion อยู่ ให้เขาเล่าว่าระบบช่วยทีมยังไง คุณพูดเรื่อง technical
- **เตรียม demo สดไว้** — เปิด LINE ส่งข้อความ 1 ข้อความ ให้เห็นบอทตอบ + ข้อมูลขึ้น Sheet ใน real-time ภาพจริงมีพลังกว่าคำพูด
- **เน้นว่ายกเลิกได้** — คนไทยกลัวการผูกมัด การบอกว่า "ยกเลิกได้ทุกเมื่อ" ลดความกังวลได้มาก
- **ถ้า MD ถามเพิ่ม** — มี Executive Summary พร้อมอยู่แล้ว (docs/13_Executive_Summary.md) เปิดให้ดูได้เลย
